# Story 5.2: Enhanced Citation Footer Component

Status: review

## Story

As a developer,
I want the raw source metadata from the LLM response to be beautifully formatted as a dedicated footer block,
so that I can easily scan and click the exact "proof" links verifying the AI's answer.

## Acceptance Criteria

1. **Given** the raw citation list output from Story 4.5
2. **When** the `ui/citations.py` helper function is called
3. **Then** it renders a visual markdown separator (`---`) at the bottom of the chat bubble
4. **And** it renders a bulleted list of citation links using sub-text styling (`font-size:0.8em; color:gray;`)
5. **And** the links are descriptive (e.g., using the `title` metadata rather than raw URLs)
6. **And** clicking a link opens the target in a new browser tab (`target="_blank"`)

## Dev Notes

- **Streamlit Link Targeting:** Standard markdown links `[text](url)` in Streamlit do not consistently open in new tabs depending on the browser environment. Using explicit HTML `<a target="_blank">` is mandatory to satisfy AC 6 and prevent the user from losing their chat state.
- **Security:** Using `unsafe_allow_html=True` is safe here because the URLs and Titles are strictly generated by our internal ingestion pipeline, not user input.
- **Architectural Constraints:** Keep this logic out of `ui/chain.py`. The chain should only return raw objects; `ui/citations.py` handles the Streamlit-specific HTML formatting.
