Messaging refers to mechanisms for interprocess communication. In [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), the messaging [cartridge](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817115/cartridge) implements multicast messaging, used to send event messages inside an Intershop Commerce Management [cluster](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817346/cluster), as well as a [framework](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2163970267/framework) for integrating external messaging systems, like JMS or JGroups.

There are four types of events that are distributed:

General application server events

Cache engine message events

ORM cache synchronization events

Tomcat cluster management events
