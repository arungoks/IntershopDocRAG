This guide lists all migrations required from Intershop Order Management 4.7 to 5.0. The document is aimed at project developers.

IOM Helm Charts 3 or newer are required to operate IOM 5.0. For more information, refer to [https://github.com/intershop/helm-charts/releases/tag/iom-3.0.0](https://github.com/intershop/helm-charts/releases/tag/iom-3.0.0).

WildFly application server was updated to version 30.0.1. Dependencies must be updated for final projects, refer to [https://www.wildfly.org/news/2023/12/05/WildFly3001-Released/](https://www.wildfly.org/news/2023/12/05/WildFly3001-Released/). Check the changes made in [IOM Project Archetype Version 2.6.0](https://github.com/intershop/iom-project-archetype/releases/tag/2.6.0) and apply these changes to your project.

This change was introduced in IOM 4.2.0, but the startup check did not work and has been fixed in this release.

The deployment now only modifies possible differences in persisted enumerations, but it throws an exception if the value has not yet been persisted and stops the deployment.

Therefore, it is necessary to write an SQL migration script to add these new values to the database, in addition to the Java definition. Check out the [iom-blueprint-project/001_ExpandedEnumerations.sql](https://github.com/intershop/iom-blueprint-project/blob/main/src/sql-config/dbmigrate/001_ExpandedEnumerations.sql) in relation to the affected `Expanded*.java`

files in [iom-blueprint-project/enums/expand/](https://github.com/intershop/iom-blueprint-project/tree/main/src/main/java/com/intershop/oms/enums/expand). For details about changes to these Java files, see [https://github.com/intershop/iom-blueprint-project/commit/a7350eb2356bac385d246ddc850f5b742c62f279](https://github.com/intershop/iom-blueprint-project/commit/a7350eb2356bac385d246ddc850f5b742c62f279).

The modification do not impact already deployed projects. However, missing `enum`

definitions in existing configuration scripts can hinder new deployments, such as creating a new test environment.

This list does not claim to be exhaustive regarding Java API, which is primarily handled via the usual Java deprecation process.
