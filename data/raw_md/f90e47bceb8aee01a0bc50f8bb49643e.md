The provider for the replication of promotion codes has been changed from the `MergeDomainSpecificStagingProcessor`

to the `DeleteAppendDomainSpecificStagingProcessor`

. This is necessary because otherwise the redeemed status of Promotion Codes is not replicated or published from EDIT to LIVE.

| Description | Change | How to migrate |
|---|---|---|
| New staging processor for promotion code staging | The processor has been changed from com.intershop.component.marketing.dbmigrate.data.7.4.5.0.staging.StagingGroupInformation.propertiesMARKETING_PromotionCode=DeleteAppendDomainSpecificStagingProcessor;Promotion/Rebate;Contains all database table storing promotion and rebate information. | The change of the configuration is part of the DBMigrate. Be aware that the processors work different. So any custom decorators will no longer work or show a different behavior. |
