An application programming interface (API) defines a contract that is intended to be used by software components to communicate with each other. An API can take different forms, like Java interfaces, command-line parameters, web service interfaces, and others. The purpose of an API is to separate the public interface, which should be stable, from the internal implementation, which may frequently change. For Intershop, the Java API is formed by the [cartridge API](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2186282051/cartridge+API). Changes in an API should be avoided to ensure compatibility and are subject to strict policies. New APIs may be added in [minor release](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2177466677/minor+release)s, existing APIs may only be changed in [major release](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2177466671/major+release)s.

In the B2X edition of [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), [business customers](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2180154362/business+customer) can use the [order](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822000/purchase+order) approval functionality to review orders before they are sent to [merchants](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2158755994/merchant). The order approval includes two checks:

Cost center approval - if an order is assigned to a [cost center](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817548/cost+center), it must first be approved by the cost center manager before being routed to the user-specific order approval.

User-specific order approval - if there is a [budget](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817090/budget) or an order spend limit defined for a [user](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822248/user), the order must pass the user-specific order approval.

If there is neither a cost center nor a user budget/order spend limit involved, no order approval takes place.

In [Enfinity Suite 6](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2155352994/Enfinity+Suite+6) B2B and procurement models, approval workflows refer to the path which a [requisition](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822049/requisition) has to pass through before being turned into a [purchase order](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822000/purchase+order).
