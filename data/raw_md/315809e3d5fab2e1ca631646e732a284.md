Deep Links and Gradle Versions

This document describes Intershop's Build Tools in different versions. Information that applies only to a certain version of the tools is clearly marked.

The document also contains deep links into the Gradle's documentation. For technical reasons they always point to Gradle version 1.8. For Build Tools version >= 2.0, please refer to the [Gradle Build Language Reference](http://www.gradle.org/docs/current/dsl/) and [Javadoc for Gradle](http://www.gradle.org/docs/current/javadoc/) version 2.0 instead. Mostly it is enough to substitute the version number in the URL, e.g., change:

[http://www.gradle.org/docs/1.8/dsl/org.gradle.api.Task.html](http://www.gradle.org/docs/1.8/dsl/org.gradle.api.Task.html)

into

Gradle itself as well as its plugins use extension objects to obtain configurations from build scripts. Besides the standard extension objects of Gradle the Intershop build tools add some Intershop specific extension objects, that are discussed in this section.

This plugin add the [Intershop extension](https://knowledge.intershop.com#Reference-GradleBuildTools-IntershopExtension) to the project. The following plugins will be applied with this plugin: base, java-base and ivy-publish. For more information see [Standard Gradle plugins](http://www.gradle.org/docs/1.8/userguide/standard_plugins.html) . This plugin will change or add some configurations:

creates the [SourceSet](http://www.gradle.org/docs/1.8/dsl/org.gradle.api.tasks.SourceSet.html) main

extend classpath with internal liberaries from the source folder *staticfiles/cartridge/lib*

changes the behavior of the compile configuration to *non transitive*

additional artifacts of an Intershop component are pre-configured in this plugin (see [Reference - Gradle Build Tools#ComponentPackage](https://knowledge.intershop.com#Reference-GradleBuildTools-ComponentPackage)).

add an Intershop component configuration for publishing (see Gradle [documentation](http://www.gradle.org/docs/1.8/javadoc/org/gradle/api/component/SoftwareComponent.html)): components.intershop

add an IVY publishing configuration for Intershop components

The workflow of tasks will be not changed by this plugin. All tasks and the workflow is implemented by standard Gradle cartridges.

This plugin applies the plugin [Reference - Gradle Build Tools#intershop-base](https://knowledge.intershop.com#Reference-GradleBuildTools-intershop-base). This plugin will change or add some configurations:

Add [Reference - Gradle Build Tools#javaSourceDir](https://knowledge.intershop.com#Reference-GradleBuildTools-javaSourceDir) of the Intershop extension as a source directory to [SourceSet](http://www.gradle.org/docs/1.8/dsl/org.gradle.api.tasks.SourceSet.html) main.

[SourceSet](http://www.gradle.org/docs/1.8/dsl/org.gradle.api.tasks.SourceSet.html) test is created and [Reference - Gradle Build Tools#javaSourceDir](https://knowledge.intershop.com#Reference-GradleBuildTools-javaSourceDir) of the Intershop extension as a source directory is added. Furthermore the the include and exclude is changed.

sourceSets { main { java { srcDir project.extensions.intershop.javaSourceDir include '**/**/*.java' } resources { srcDir project.extensions.intershop.javaSourceDir include '**/**/**' exclude '**/**/*.java' } } test { java { srcDir project.extensions.intershop.javaSourceDir include 'tests/**/**' exclude 'tests/server/**/*.java' exclude 'tests/unit/**/*.java' } } }

The classpaths of the compile configuration of `main`

and the runtime configuration `test`

are extended.

The following configurations are added: `testRuntime`

, `testCompile`

, `source`

and `javadoc`

.

This plugin adds also some tasks: `test`

and `jar<SourceSet name>`

.
