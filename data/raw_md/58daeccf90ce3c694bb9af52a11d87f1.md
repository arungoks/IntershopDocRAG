Persistent objects (POs) are Java objects used to write and read data to/from the [database](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729187/database) ("[ORM](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2167931836/ORM) beans"). They are persistent implementations of [business object](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2158101810/business+object)s.

In the [layer](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2180161411/layer)ed software architecture, the persistent object layer groups the [persistent object](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2167931462/persistent+object)s, which are used to write and read data to/from the [database](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729187/database) (see [ORM](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2167931836/ORM)). In [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), the persistent object layer is implemented in the [ORM engine](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214735737/ORM+engine).
