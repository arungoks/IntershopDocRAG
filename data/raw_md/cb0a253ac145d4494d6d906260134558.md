The PWA can be monitored using [Prometheus](https://prometheus.io).

Activating the feature Prometheus for the nginx or SSR container exposes metrics on port 9113 on the `/metrics`

URL.

The SSR container also offers the configuration parameter `METRICS_DETAIL_LEVEL`

to control the granularity of collected metrics:

Value `DEFAULT`

:

Value `DETAILED`

:

After activating the features on both containers, add tasks to prometheus:

```
scrape_configs:
- job_name: 'pwa-nginx'
scrape_interval: 5s
dns_sd_configs:
- names:
- 'tasks.pwa-nginx'
type: 'A'
port: 9113
- job_name: 'pwa-ssr'
scrape_interval: 5s
dns_sd_configs:
- names:
- 'tasks.pwa-ssr'
type: 'A'
port: 9113
```

To get started, you can import our example [Grafana Dashboard](https://raw.githubusercontent.com/intershop/intershop-pwa/develop/docs/guides/prometheus-monitoring-dashboard.json):

Note

The example dashboard is supposed to be used with `METRICS_DETAIL_LEVEL=DETAILED`

only.

Warning

We recommend to switch off the SSR container health-check and define alerts in Grafana instead.
