Generally, a merchant is a business actor who trades in goods or services produced or delivered by others, in order to earn a profit.

From Intershop's perspective, a merchant is the [sales organization](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153809331/sales+organization) that uses Intershop's software to drive its e-commerce initiative. With respect to the system operation, a merchant can either run the software on its own premises, or contract a hosted environment as a [tenant](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2158760205/tenant).

Messaging refers to mechanisms for interprocess communication. In [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), the messaging [cartridge](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817115/cartridge) implements multicast messaging, used to send event messages inside an Intershop Commerce Management [cluster](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817346/cluster), as well as a [framework](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2163970267/framework) for integrating external messaging systems, like JMS or JGroups.

Microservices are an architectural pattern in computer software design, in which complex software is composed of small, independent processes that act together in order to fulfill a common goal. Microservices are self-contained units of [functionality](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2152516365/feature), which are largely decoupled, do a specific (small) task and communicate with each other using technology-agnostic protocols. This allows for a modular software design, leveraging [continuous refactoring and releasing](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214727367/continuous+integration).

Multi-data center support refers to the possibility to distribute [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management) clusters over multiple [data center](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2167931527/data+center)s in an active-active scenario. Each data center hosts one or more Intershop [cluster](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817346/cluster)(s) consisting of multiple [Intershop Web Server](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2165834404/Intershop+Web+Server)s, multiple [Intershop Application Server](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2165834397/Intershop+Application+Server)s, at least one [Intershop Shared Files](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2165834401/Intershop+Shared+Files) instance and a [database](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729187/database) instance. The Intershop clusters are synchronized with each other to support session failover between the data centers. Each cluster hosts several [site](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2163974126/site)s, whereby a site is only active in one data center. Oracle Streams is set up and used to synchronize transactional data between the databases in the data centers.

The Node Manager starts, stops and (in case of abnormal process termination) restarts application server processes. In addition, it provides the communication interface for [Cluster Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2200669545/Cluster+Management) that is used for remote control purposes.
