In software engineering, VD is a part of the [IAD](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2163976934/IAD) and comprises all elements that make up the [user interface](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2162594514/user+interface). Aspects of the visual design include branding and visual emphasis, which are achieved with placement, color, typography, iconography and images. [view context](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2190216347/view+context) | In the [WCM](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2192548835/WCM) of [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), a view context is a means to define which content entry point ([page](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2190216300/page) or [include](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2190216337/include)) is used to render a [business object](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2158101810/business+object) ([product](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160821932/product) or [category](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817282/category)) for a given context (like catalog browsing or checkout). That is, it allows for rendering an object in specific ways in different areas of the [storefront](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2178600227/storefront), according to conditions like, for example, the individual object, given constraints of the object (based on the type of object), the relation to other pages (e.g., top-level category or sub-category), fallback view. |
