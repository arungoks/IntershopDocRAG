In software, a dashboard is generally a real-time [user interface](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2162594514/user+interface) that combines easy-to-read information about a [user](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822248/user)'s tasks, business KPIs, etc.

In [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), a dashboard is a customizable set of views ([widget](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2190214639/widget)s) that display certain data and allow to navigate to the corresponding [back office](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2159969605/back+office) area.

A data center is a facility used to house computer systems and associated components, such as telecommunications and storage systems. It generally includes redundant or backup power supplies, redundant data communications connections, environmental controls (e.g., air conditioning, fire suppression) and various security devices.

[Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management) clusters can be distributed over multiple data centers in an active-active scenario. Each data center hosts one or more Intershop [cluster](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817346/cluster)(s) consisting of multiple [Intershop Web Server](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2165834404/Intershop+Web+Server)s, multiple [Intershop Application Server](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2165834397/Intershop+Application+Server)s, at least one [Intershop Shared Files](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2165834401/Intershop+Shared+Files) instance and a [database](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729187/database) instance. The Intershop clusters are synchronized with each other to support session failover between the data centers. Each cluster hosts several [site](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2163974126/site)s, whereby a site is only active in one data center. Oracle Streams is set up and used to synchronize transactional data between the databases in the data centers.

Data replication is a process to transfer large amounts of data from a [source cluster](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822129/source+cluster) to a [target cluster](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2160822228/target+cluster).

As a typical scenario, one could first update any [storefront](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2178600227/storefront) data (like product data) and other settings in an "[editing system](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2179399785/editing+system)" and then transfer this information to a "[live system](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2179399780/live+system)". This mechanism allows to develop and maintain large content in the background without significant disturbances to the production system.

The mechanism for transferring individual [business object](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2158101810/business+object)s in an ad-hoc manner is called [object replication](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2178023704/object+replication) (developer and administrator perspective) or [publishing](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2178023699/publishing) (shop manager perspective).

With [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management), the relational database system is used to store data whenever a transactional access or search capability is required. As a rule of thumb, all business data and some of the system configuration data is stored in the relational database system. Access to the database system happens primarily via the object-relational mapping mechanism of the [ORM](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2167931836/ORM) engine, which internally uses the native client drivers to access the database server.

Currently, Intershop Commerce Management support Oracle database systems only.

DBInit is a tool for initially creating the database schema objects and for preparing the [database](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729187/database) content of one or more [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management) [cartridge](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2153817115/cartridge)s. It is not designed to migrate the database content and structure (see [DBMigrate](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729194/DBMigrate)).

DBMigrate is a tool used to migrate the [database](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2214729187/database) content and the database structure of an [Intershop Commerce Management](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2217982072/Intershop+Commerce+Management) installation from a defined previous [release](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2159968461/release) to an installed current release. It defines one global migration process, which can be executed in different environments. DBMigrate reuses some recurring migration steps (e.g., table creation), supports different development teams to create their own migration processes, and can minimize the effort to upgrade the development environment.

DBPrepare is a tool for creating and updating the database schema objects and for preparing required database content of one or more cartridges. The DBPrepare tool recognizes prepared or up-to-date cartridges and their declared preparer steps. As a result of stored execution protocol for each cartridge, DBPrepare only needs to execute new preparer steps, resulting in a significantly reduced execution time. Additionally, DBPrepare initializes new cartridges, so explicit initialization of new cartridge is no longer required.

DBPrepare uses the declaration files of DBinit and DBmigrate and replaces the functionality of DBInit and DBMigrate.

A direct custom attribute (DCA) is an extensible object attribute that is stored directly in a column in the table of the extensible object. This is achieved by a special DCA configuration for the [ORM](https://knowledge.intershop.com/kb/go.php/a/ISGLOS/pages/2167931836/ORM) engine.
