Welcome to Intershop Commerce Management 13.

This Public Release Note contains information about the latest features of Intershop Commerce Management. In addition, it serves as a quick reference for the latest documentation.

With this release, storefront REST API performance has improved by 10% for average requests.

The Product and Catalog Management REST API for the back office has been enhanced by new functionality that ensures more efficient data transfer and simplifies catalog maintenance tasks.

For details about the new functionality, see [What's New in ICM 13](https://knowledge.intershop.com/kb/go.php/a/ENFDEVDOC/pages/49157242887/What+s+New+in+ICM+13)*.*

Updated Apache tika-core library to version 3.2.3 to fix the critical XXE vulnerability ([CVE-2025-66516](https://nvd.nist.gov/vuln/detail/CVE-2025-66516))

Proxy product feature toggle

The Product Information Management (PIM) includes a proxy product feature. A toggle to enable or disable this feature has been added.

See [Guide - 13.4.0 - Proxy Product Feature Toggle](https://knowledge.intershop.com/kb/go.php/a/ENFDEVDOC/pages/50138185747/Guide+-+13.4.0+-+Proxy+Product+Feature+Toggle) for details.

Data replication - enhanced *tolerateFileFailure* toggle

In the mass data replication process, a toggle to tolerate some failures has been added to the *SimpleFileSystemStagingProcessor*.

See [Guide - 13.4.0 - Data Replication - Enhanced tolerateFileFailure Toggle] for details.

Product list performance improvement toggles

Two properties have been added that you can configure to limit the data retrieved and displayed in the back office product detail view to improve performance.

See [Guide - 13.4.0 - Toggles for Product List Performance Improvement in the Back Office UI](https://knowledge.intershop.com/kb/go.php/a/ENFDEVDOC/pages/50152013920/Guide+-+13.4.0+-+Toggles+for+Product+List+Performance+Improvement+in+the+Back+Office+UI) for details.

Improved accessibility of *icm_as* e-mail templates to comply with WCAG standards

Added promotion conditions for recurring orders

[Reference - Wishlist / Order Template REST API 1.3.0](https://knowledge.intershop.com/kb/go.php/a/ENFDEVDOC/pages/48863346718/Reference+-+Wishlist+Order+Template+REST+API+1.3.0): Renamed the former *Wishlist REST API* to *Wishlist / Order Template REST API*.

This change is purely a renaming and does not involve any technical modifications.

The new name better reflects the API's dual purpose: managing wishlists and creating order templates based on the wishlist functionality.
