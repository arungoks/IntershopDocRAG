# Configuring Audit Report

The Audit Report Configuration view presents a set of filters for configuring a report. By default all object events that have occurred within the previous two years are included in the report. The following table provides a description of each configurable filter.

| Filter | Description |
|---|---|
| Context | This filter is available at the organization and channel level. It allows you to select the management contexts from which events/object activity is returned. |
| User | This filter allows you to configure a report that returns the activity of the selected users. If a particular management context is selected, only the users associated with that management context are shown and able to be selected. |
| Object Type | This filter allows you to configure a report that returns only the selected objects. If a particular management context is selected, only the objects associated with that management context are shown and able to be selected. |
| Transaction Type | This filter allows you to configure a report that returns only the selected events. |
| Date Range | The date picker allows you to set the start and end dates for the report configuration. |

Note:

By default each filter is set to Select All. You can manually select multiple values, however, at least one value must be selected. Otherwise the system will revert to the default setting.
