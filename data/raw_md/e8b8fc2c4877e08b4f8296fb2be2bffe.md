# Locking Includes

Before editing an include, you should lock the include. Locking prevents the include from being edited by other users in parallel. The include lock can be released manually. It is released automatically after the configured lock interval has expired. For details about how to configure the include lock interval, see [Defining Content Lock Period](https://knowledge.intershop.com/oh/go.php/icm/master/icm_main_en/operation_maintenance/task_defining_content_lock_period.html).

Basically, there are two ways to lock includes: either directly from the include list, or via the include detail view.

## Locking Includes in List View

-
Select the management context from the context selection box, then select Content | Includes.
This opens the Include list view.
-
Click the icon in the row of the include(s) you intend to edit.
An error message appears if the include is currently edited by another user.

## Locking Includes in Detail View

-
Select the management context from the context selection box, then select Content | Includes.
This opens the Include list view.
-
Click the name of the include you intend to edit.
This opens the Content tab of the detail view.
-
Click the icon at the dialog top.
An error message appears if the include is currently edited by another user.
